Abstract

This report presents the design and development of an Agentic AI Mail Assistant, a system that leverages Large Language Models (LLMs) and APIs to automate email management. The purpose of this report is to document the complete workflow of the project — from conception and methodology to implementation and results — while highlighting the challenges and solutions encountered during development.

  • The project focuses on simplifying the way users interact with their inboxes. Using the Gmail API, the assistant retrieves the latest important emails, processes their contents, and stores them in structured JSON files.
  • With the help of an LLM (initially LLaMA 3 via Ollama, later optimized using the Gemini API), these emails are summarized into concise, meaningful insights.
  • A user-friendly Streamlit interface then allows users to view summaries, expand to see the full email, generate AI-powered draft responses, and send replies directly using the Gmail API.

This integration of retrieval, summarization, and automated response generation demonstrates how agentic AI systems can reduce cognitive load, save time, and enhance productivity in everyday tasks.

Introduction

Email is one of the most widely used forms of digital communication, but managing it effectively continues to be a major challenge. Overflowing inboxes, lengthy email threads, and repetitive responses often consume valuable time and attention, reducing overall productivity. As organizations and individuals increasingly rely on email for professional and personal communication, the need for intelligent systems to simplify inbox management has grown.

This project addresses that challenge by developing an Agentic AI Mail Assistant — a system designed to automate and streamline email handling using the power of Large Language Models (LLMs). The assistant performs three core tasks:

  • Retrieval - Fetching the most recent important emails using the Gmail API.
  • Summarization - Condensing long or complex messages into concise summaries through an LLM.
  • Response Generation - Providing AI-powered draft replies that users can review, modify, and send directly via the Gmail API.

The project began with experiments using a local LLaMA 3 model via Ollama, which offered strong capabilities but limited speed. To improve responsiveness, the system was later optimized with the Gemini API, delivering faster inference and more reliable results. On the front end, a simple yet effective Streamlit-based interface was built to give users direct control over their inbox.

Through this integration of retrieval, summarization, and intelligent response generation, the Mail Agent demonstrates how agentic AI systems can be applied to real-world productivity challenges, reducing manual effort while keeping the user in control.

Problems & Solutions

Throughout the development of the Agentic AI Mail Assistant, several technical challenges were encountered. Each challenge provided valuable learning opportunities and led to innovative solutions that improved the overall system.

1. Slow Performance with Local LLM (LLaMA3 via Ollama)

Problem

Running LLaMA3 locally was resource-intensive and produced delays in summarization and draft generation.

Solution

Shifted to Gemini API, a cloud-based LLM, which significantly improved response speed and overall efficiency.

2. Repeated API Calls to Gmail

Problem

Fetching and summarizing emails directly from Gmail API every time caused slowdowns and unnecessary requests.

Solution

Designed a two-step JSON workflow:
• Store raw emails in raw_emails.json.
• Summarize and store them in summarized_emails.json.
Frontend retrieves data from JSON, minimizing Gmail API usage.

3. UI Responsiveness & Usability

Problem

Early versions of the UI were plain and lacked smooth interactions, making the project less user-friendly.

Solution

Introduced Streamlit for interactivity and GSAP animations in the report webpage to improve engagement and navigation.

4. Secure Email Sending

Problem

Directly sending emails posed risks without proper authentication.

Solution

Integrated Gmail API with OAuth 2.0, ensuring secure, reliable sending of emails.

5. Draft Generation Accuracy

Problem

Initial drafts were too verbose or irrelevant.

Solution

Optimized prompts for the LLM to strictly generate concise, context-aware replies.
